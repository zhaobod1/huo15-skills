# 火一五设计方向顾问技能 v1.0

> 多方向设计提案生成 — 青岛火一五信息科技有限公司

---

## 一、触发场景

当用户要做一个页面/产品/品牌，但**没明确美学方向**时：

- "帮我选个设计方向"
- "做三个风格对比"
- "这个产品应该做成什么风格"
- 或在 `huo15-openclaw-frontend-design §三` 被用户选"让你决定"时自动触发

**产出**：3 个反差方向的 Junior pass（缩略图级 HTML 代码段）+ 五维对比矩阵 + 推荐方向。

---

## 二、设计哲学库（20 条可引用）

### 2.1 极简主义派（5 条）
1. **原研哉（Kenya Hara）** — 白、空、无印良品式的留白美学
2. **Dieter Rams 十诫** — 少即是多，诚实、长寿、环保
3. **Swiss Design / 国际主义** — 网格、无衬线、左对齐
4. **Apple 后乔布斯** — 超大字、超大图、一屏一个重点
5. **Stripe 极简科技** — 渐变 + sans-serif + 大量白

### 2.2 编辑杂志派（4 条）
6. **NYT / 纽约客** — 衬线、纵向栅格、引号做装饰
7. **Monocle 杂志** — 深色衬线 + 米色底 + 分栏
8. **Pentagram 平面** — 超大字排版 + 强烈对比
9. **Hoefler & Co 字体中心主义** — 字体本身就是视觉

### 2.3 前卫实验派（4 条）
10. **Sagmeister 观念先锋** — 手写、大胆、打破规则
11. **David Carson 破坏性排版** — 故意错位、字符切割
12. **Brutalist Web Design** — 粗黑、原始、反精致
13. **Y2K / Vaporwave 复古未来** — 霓虹、CRT、像素

### 2.4 东方/有机派（4 条）
14. **东方禅意** — 枯山水、空寂、留白 80%
15. **日本民艺（Mingei）** — 手工感、温润、不对称
16. **Field.io 动态几何** — 数字生成艺术
17. **Organic / 有机自然** — 手绘、暖色、柔边

### 2.5 信息/功能派（3 条）
18. **Tufte 数据可视化** — 数据墨水比、无废装饰
19. **Material Design（新版）** — 分层、拟物、明确反馈
20. **Bauhaus 功能主义** — 形式追随功能、三原色

## 三、3 方向生成法

### 3.1 方向选取规则

**不要选 3 个相似方向**。必须组合：

- **1 个保守方向**（BOLD-MINIMAL 或 EDITORIAL，不出错的经典）
- **1 个反差方向**（BRUTALIST / RETRO-FUTURE / 东方禅意，有个性但有风险）
- **1 个中间调和方向**（ORGANIC / 新 Material，平衡美学和可用性）

### 3.2 每个方向必给 5 件东西

```
### 方向 N：<流派名>
- **一句话定位**：[ 为谁做 + 什么感觉 ]
- **核心字体**：display + body 组合
- **主色 + 强调色**：含 oklch 值
- **关键元素**：3 个会被用户记住的视觉元素
- **Junior pass HTML**（150 字符内的缩略代码段，表明布局）
```

### 3.3 五维对比矩阵

| 维度 | 方向 1 | 方向 2 | 方向 3 |
|------|--------|--------|--------|
| 美学震撼 | ★★★☆☆ | ★★★★★ | ★★☆☆☆ |
| 可用性 | ★★★★★ | ★★★☆☆ | ★★★★☆ |
| 品牌辨识 | ★★☆☆☆ | ★★★★★ | ★★★☆☆ |
| 实现成本 | 低 | 高 | 中 |
| 风险 | 低 | 高（可能不被客户接受） | 中 |

### 3.4 推荐表态

**必须**给出一个推荐（不能骑墙）：

```
### 推荐
选 **方向 N**，理由：[结合用户的目的、受众、约束的三句话推理]

### 次选
方向 M，适用场景：[如果方向 N 不被接受的退路]
```

## 四、工作流

### 阶段 1 · 需求分解
- 复述用户的：**目的 / 受众 / 约束 / 时间预算**
- 如果以上任一缺失，先用一轮问答补齐

### 阶段 2 · 方向筛选
从 20 条设计哲学库中挑 3 条（按 §3.1 规则组合）

### 阶段 3 · Junior pass 生成
每个方向出一段 150 字符内的 HTML 缩略代码 + 截图说明

### 阶段 4 · 对比 + 推荐
按 §3.3 + §3.4 出报告

### 阶段 5 · 移交
用户选定方向后，把该方向的 brief 交给 `huo15-openclaw-frontend-design` 做 Full pass。
