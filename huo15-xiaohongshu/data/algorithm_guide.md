# 小红书算法指南（2025-2026）

> 基于行业公开资料整理，供创作策略参考。

## CES 评分公式

**CES（Community Engagement Score）** 是小红书推荐系统的核心指标，决定笔记能否进入下一级流量池。

```
CES = 点赞 × 1 + 收藏 × 1 + 评论 × 4 + 转发 × 4 + 关注 × 8
```

### 权重解读

| 行为 | 权重 | 策略 |
|------|------|------|
| 关注 | ×8 | 最高权重 — 内容的核心目标是让读者关注 |
| 转发 | ×4 | 高权重 — 转给朋友/闺蜜 = 最真实的推荐 |
| 评论 | ×4 | 高权重 — 埋钩子、留槽点、造争议 |
| 收藏 | ×1 | 中权重 — 实用合集、清单、攻略 |
| 点赞 | ×1 | 最低权重 — 顺手点赞，不代表真价值 |

> ⚠️ 2026 年更新：更侧重**互动质量**。长评论 > 短评论，深度阅读 > 随手刷，截屏/保存等高质量互动权重更高。单纯点赞加分占比持续降低。

## 冷启动机制

```
发布 → 审核 → 收录至搜索页 
     → 分配初始流量池（200-500 曝光）
     → CES 评分决定是否进入下一级流量池
     → 循环：更多曝光 → 更高 CES → 更大流量池
```

### 关键节点

1. **审核阶段**：标题、封面、正文、评论、话题标签提取关键词
2. **冷启动**：200-500 次曝光测试，看 CES 能否达标
3. **下一级**：如 CES 达标，进入千级流量池，继续检验
4. **滚雪球**：经过 3-5 轮筛选，进入万级/十万级池

## 内容质量要素（2025）

| 要素 | 权重 | 说明 |
|------|------|------|
| 内容原创率 | **35%** | NoteLLM 模型 + SSD 算法识别原创度 |
| 信息密度 | 高 | 图文笔记需含足够干货/深度信息 |
| 用户停留时长 | 关键信号 | 算法通过兴趣标签延长使用时长 |
| 兴趣标签匹配 | 基础 | 根据内容标签匹配兴趣用户推送 |

### NoteLLM 模型

2025 年全面启用，冷启动效率提升 3.58%。核心：
- 多模态理解：文字+图片联合分析
- 标签精准匹配：从内容自动提取标签，推给最匹配的用户
- 原创度检测：识别洗稿、搬运、AI 生成腔

## 创作策略建议

### 1. 内容运营

- **首段即高潮**：前 13 字决定搜索展示中的点击率
- **信息密度**：一篇文案至少 3-5 个可收藏的干货点
- **图文并茂**：文字+图片联合优化，NoteLLM 分析两者相关性
- **原创优先**：原创率权重大幅提升，禁止洗稿/搬运

### 2. 互动引导

- **评论埋钩子**：在文中留槽点/争议/开放问题，引导评论
- **长评论诱导**：写深度分析文，自然引发讨论
- **转发场景**：写「建议艾特闺蜜/男朋友」类的指向性内容
- **收藏理由**：清单、合集、攻略天然高收藏率

### 3. 标签策略

- **2 泛 + 2 垂 + 1 长尾**：2 个泛话题（覆盖面广）+ 2 个垂直话题（精准）+ 1 个长尾话题（竞争小）
- **高频标签**：观察热榜话题，及时跟进
- **品牌标签**：自有品牌/活动标签必加

### 4. 发布节奏

- **每天 ≤ 2 篇**，间隔 ≥ 2 小时
- **黄金时段**：早 8-9 点 / 午 12-13 点 / 晚 18-22 点
- **连续测试**：同一主题隔天发第二篇，看 CES 对比
