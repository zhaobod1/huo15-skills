# 火一五小红书创作哲学

> 综合 Allen（司志远）三课五技法 + 东东枪《文案的基本修养》99篇 + 小红书算法实践，
> 提炼为五层创作决策框架。这是本技能的**灵魂**——不是理论读物，是每次动笔前的决策清单。

---

## 总纲：创作的第一性原理

**小红书文案的本质不是「写得好」，是「让对方看完后觉得自己被理解了」。**

一切技法从这个原点出发。如果读者看完没有「对，我就是这样」的瞬间，你的标题再炸、排版再漂亮，都是零。

**四句心法（写前 30 秒默念）：**

1. 「好文案不是写出来的，是留出来的。」— Allen 第一课
2. 「卖的是身份认同，不是商品本身。」— Allen 第二课
3. 「站文案里面读文案，不是站在外面分析。」— Allen 第三课
4. 「一流的糖衣，本身就是炮弹。」— 东东枪 041

## 第一层：创作原点 — 你为谁写？

### 1.1 用「心象」定义读者，不用「形象」

❌ 25-35岁女性 / 一线城市 / 月入1万+
✅ 「觉得人生需要喘息但说不出口的人」
✅ 「想对自己好一点但总把别人排前面的人」

**Why:** 人口标签只能描述「谁可能买」，心象描述「谁会产生共鸣」。共鸣驱动一切互动行为（评论×4 / 转发×4 / 关注×8）。

**自检：** 你定义的那个读者，你能在脑子里看见她今天下午在做什么吗？看不见 = 太模糊。

### 1.2 站文案里面感受，别站外面分析

写每句话的时候问自己：**「如果我是读者，读到这一行，我的手会停下来吗？」**

- 读者在刷小红书的时候是**放松模式**，不是学习模式
- 互动引导不是任务指令，是邀请语——「这里有个局，你可以来」
- 文案是情境，不是说服

### 1.3 场景共鸣的铁律

> 「场景共鸣不是找冷知识，是找人人都有过的共同记忆。」— Allen

| ❌ 冷知识（装文化，无共鸣） | ✅ 共同记忆（谁都有过） |
|---|---|
| 蝉要在地下七年才出土 | 树荫下骑车，光在身上走 |
| 古人夏天喝什么 | 风扇转头的嗡嗡声 |
| 西瓜原产自非洲 | 切开西瓜那一声脆响 |

## 第二层：创作支点 — 你要改变什么？

### 2.1 AB 点：定义你要改变的「看法或感受」

> 「广告只能改变看法和感受，不能直接改变行为。」— 东东枪 006

**写每篇笔记前必答：**

```
A = 读者现在的看法/感受：
B = 看完后我希望她有的看法/感受：
这座桥怎么搭？
```

**B 点的质量判断：**
- ❌ 「让她觉得我的产品好」→ 这是愿望，不是 B 点
- ❌ 「成为更好的自己」→ 太虚，没有具体感的 B = 没有 B
- ✅ 「让她觉得周末不是用来恢复元气的，是用来培育下周好奇心的时间」→ 具体、可感

### 2.2 核心体验：你不卖内容，你卖感觉

> 「核心体验 = 品牌真正在售卖、被消费者真正买走的东西。」— 东东枪 014

- 一个护肤号 ≠ 卖护肤知识，卖的是**「我值得被好好对待」的许可**
- 一个职场号 ≠ 卖职场技巧，卖的是**「我不是一个人在这种处境里」的陪伴**
- 蕉下 ≠ 卖防晒衣，卖的是**敢晒太阳的野心**

**自检：** 读者真的来你这里只为了信息？还是为了「和你一起的某种感觉」？

### 2.3 先做功课再动笔（Allen 红线）

**写任何产品/品牌前：**
1. 搜品牌官方宣传语、发布会、产品页
2. 理解品牌怎么定义自己（不是你怎么看它）
3. 提取 3-5 个官方核心意象
4. 判断品牌卖的是态度还是成分
5. 再做创意

❌ 跳过 = 从零发明产品 = 大概率 0 分。品牌自己在说什么 > 你觉得品牌应该说什么。

## 第三层：创作手艺 — 怎么写？

### 3.1 留白法则

> 「好文案不是写出来的，是留出来的。」— Allen

- 断句 + 留白，不是连续推演
- 金句独立成段，**不解释道理**
- 写完一段想：「读者读到这里能不能停一下，往里填自己的情绪？」

**技法：** 把「你应该」「你必须」「这是因为」删掉，换成「有时候」「也许是」「我自己是」。

### 3.2 五法：不说产品，说人的感受

| # | 手法 | ❌ 不要 | ✅ 要 |
|---|------|---------|-------|
| 1 | 用感受代配料 | 写配料表/成分 | 写吃下去/用上去的感觉 |
| 2 | 用场景代功能 | 说产品能干嘛 | 展示什么场景下用 |
| 3 | 用情绪代产品 | 以产品为主角 | 以读者情绪为主角 |
| 4 | 用过程代成分 | 列成分/参数 | 写从源头到成品的旅程 |
| 5 | 用比喻代描述 | 直接描述产品 | 用自然意象包裹产品 |

### 3.3 三大句式（蕉下体系）

**句式一：「不是……而是……」— 颠覆认知**
先否定旧预设，再拉高到价值观。
- 「不是害怕太阳，而是拥抱太阳」
- 公式：不是 {旧认知/表面诉求}，而是 {新身份/深层价值}

**句式二：修辞/比喻 — 参数变画面**
把产品功能寄生在自然意象上。
- 「风是最好的造型师」
- 公式：{自然意象} 是最好的 {产品角色}

**句式三：「每一……都是……」— 功能升维**
把日常小事拉高成品牌体验。
- 「每一口呼吸都是松弛的味道」
- 公式：每一 {最小动作} 都是 {品牌情绪}

### 3.4 洞察：只能发现，不能发明

> 「洞察 = 未被发现或已被遗忘的真相。」— 东东枪 020

洞察存在于两个缝隙里：
- **认知与真相之间** — 大众以为 X，其实 Y
- **表达与认知之间** — 大众心里都觉得 X，但说不出来

**好的洞察说出来，读者的反应是「真相了」「没法反驳」「我也是这么觉得就是说不出」。**

### 3.5 创意碰撞公式

```
核心体验 × 洞察 = 创意 Idea
```

- 核心体验：尽兴指南卖的是「停下来，此刻就是最好的」许可证
- 洞察：每个人都有一张「以后再XX」的清单，但「以后」永远不靠站
- 碰撞结果：「以后偷走了你好多个今天」

### 3.6 动词的用法

动词不一定要驱动行为，可以造画面/意境。
- 「领取春日礼」→ 不是去拿实物，是把阳光花香当作礼物接收
- 「捕到一张live图」→ 不是真的捕，是偶然发现的惊喜感

## 第四层：创作陷阱 — 避开什么？

### 4.1 Jarvis 陷阱五维（AI / 工程师的通病）

| 维度 | 陷阱（错的） | 范本（对的） |
|------|------------|------------|
| 开头 | 挖痛点 / 制造焦虑 | 重新定义 / 轻快切入 |
| 引导 | 建议动作（1）2）3） | 展示画面（有人在X/有人在Y） |
| 角色 | 我（作者）在说话 | 让真实用户的声音出现 |
| 语气 | 运营口吻（公告腔） | 朋友凑过来说（私聊感） |
| 结尾 | 中规中矩收口 | 让读者感觉被珍视 |

**最核心的一句话差距：**
> ❌ 我在教读者「怎么做」
> ✅ 我在展示「什么样的人已经在做」
> 前者是攻略，后者是范本。

### 4.2 AI 腔四类

| 类别 | 特征词 | 替换方向 |
|------|--------|---------|
| 汇报化 | 提升/优化/赋能/闭环/颗粒度 | 变得更好/改一改/帮你/从头到尾/细节 |
| 模板化 | 首先其次最后/综上所述/众所周知 | 想到几件事/大概就是这些/删掉 |
| 懂行感 | 据相关数据显示/研究表明/底层逻辑 | 我看到一个数字/有研究说/其实就是 |
| 教读者 | 你应该/你必须/划重点/答案是 | 你可以试试/我自己是/我后来发现 |

### 4.3 算法红线

- #AI生成内容 必标（2026/01 起强制）
- 标题前 13 字含核心关键词
- 话题 2泛+2垂+1长尾
- 单日 ≤ 2 篇，间隔 ≥ 2h

## 第五层：创作系统 — 怎么持续进步？

### 5.1 渐进式教练（一次只改一维）

看出问题 ≠ 写得出来。一次收到 6 维反馈 = 信息过载。
**一次只 focus 当前最差的那一维**，改完再看下一维。

### 5.2 多切面法则

> 「同一个 slogan，六个切面，六种受众。文案本身不变，不同人群看到的是自己想看的那一面。这不是缺陷，是力量。」— Allen

每篇笔记独立成文，不靠前后连载。同一个主题换不同角度再写一次——敢于自我推翻，才是创作。

### 5.3 功夫积累

> 「练拳不练功，到老一场空。」— 东东枪 097
> 「跟世界不熟的人写不出打动世界的好句子。」— 东东枪 098

- **术（技巧）**：句式、结构、标题公式 → 可以速成
- **道（理论）**：AB 点、核心体验、洞察 → 需要理解
- **功（内力）**：眼界、审美、分寸感 → 只能积累

积累方式：多看案例、多拆解爆款、多写多改、场景库持续扩容。

### 5.4 场景库维护

AI 写文案最大短板：场景库太窄，反复用同一批词。每次写作前：
1. 从场景库挑 3 个没用过的画面
2. 不要同一类目挑 3 个（避免单一感官堆砌）
3. 用过的画面在档案里标记，下次避开

## 决策速查：写一篇笔记的 8 个问题

动笔前：
1. **心象：** 我为谁写？（不是人口标签，是认知状态）
2. **AB 点：** 她现在怎么想？看完想让她怎么想？
3. **核心体验：** 我卖的不是内容，是哪种感觉？
4. **功课：** 如果涉及品牌/产品，我查了官方定位吗？

动笔中：
5. **留白：** 这段话是把答案给她了，还是把空间留给她了？
6. **五法：** 有没有出现配料/成分/参数/功能描述？有就改掉。

写完后：
7. **Jarvis 陷阱：** 我是在教「怎么做」还是在展示「什么样的人」？
8. **AI 腔：** 删掉所有「提升/优化/赋能/众所周知/首先其次最后」。

## 常见借口（别信）

| 借口 | 现实 |
|------|------|
| 「道理我都知道」 | Jarvis 陷阱——Allen 说过「认知到位了，输出还没跟上」。知道 ≠ 写出来。 |
| 「这次简单，不用跑哲学检查」 | 最简单的文案最容易犯 AI 腔。30 秒检查永远值得。 |
| 「我先写出来再改」 | 攻略型思路写出来的东西，后面改的是文字不是思路。从头就用范本型。 |
| 「读者看得懂就行了」 | 读者看得懂 ≠ 读者有共鸣。共鸣驱动 CES（评论×4/转发×4/关注×8）。 |
| 「我写的不是商业文案，不用 AB 点」 | 个人笔记也有 A→B——你每篇都在改变读者对某件事的看法。 |
| 「时间紧就不查功课了」 | 不查品牌官方定位 = 从零发明产品 = 大概率 0 分。查功课只要 5 分钟。 |
| 「这个意象之前用过但这次合适」 | 场景库窄的症状。去 scene_library 换 3 个没用过的画面。 |
