# 辉火云企业版 v18.8.0 部署文档

## 一、环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| OS | Ubuntu 22.04 LTS / CentOS 8 | x86_64 |
| Python | 3.11+ | 推荐 3.11.7 |
| Node.js | 20+ | 用于前端构建 |
| PostgreSQL | 15+ | 主从复制 |
| Redis | 7+ | 缓存与队列 |
| Nginx | 1.24+ | 反向代理 |
| 内存 | ≥ 16 GB | 4C8G 起步，推荐 8C16G |
| 磁盘 | ≥ 200 GB SSD | 数据库独立盘 |

---

## 二、依赖安装

```bash
# Ubuntu
apt-get update
apt-get install -y python3.11 python3.11-venv nodejs npm \
                   postgresql-15 redis-server nginx

# 创建数据库
sudo -u postgres createdb huo15
sudo -u postgres psql -c "CREATE USER huo15 WITH PASSWORD 'CHANGE_ME';"
sudo -u postgres psql -c "GRANT ALL ON DATABASE huo15 TO huo15;"
```

## 三、部署步骤

### 步骤 1 — 拉取代码

```bash
git clone https://github.com/huo15/huo15-platform.git /opt/huo15
cd /opt/huo15
git checkout v18.8.0
```

### 步骤 2 — 后端部署

```bash
python3.11 -m venv /opt/huo15/venv
source /opt/huo15/venv/bin/activate
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
vim .env   # 填 DB_URL / REDIS_URL / SECRET_KEY 等

# 数据库迁移
python manage.py migrate
python manage.py collectstatic --noinput

# 启动（systemd 推荐）
systemctl enable huo15-backend
systemctl start huo15-backend
```

### 步骤 3 — 前端构建

```bash
cd /opt/huo15/frontend
npm ci
npm run build
# 产物在 dist/，由 Nginx 直接 serve
```

### 步骤 4 — Nginx 配置

```nginx
upstream huo15_backend {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name app.huo15.com;

    location / {
        root /opt/huo15/frontend/dist;
        try_files $uri /index.html;
    }

    location /api/ {
        proxy_pass http://huo15_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 步骤 5 — HTTPS

```bash
apt-get install certbot python3-certbot-nginx
certbot --nginx -d app.huo15.com
```

## 五、回滚

```bash
# 1. 停服
systemctl stop huo15-backend nginx

# 2. 数据库回滚（如果有 schema 变更）
python manage.py migrate <app_name> <previous_migration>

# 3. 切回旧代码
cd /opt/huo15
git checkout v18.7.0

# 4. 重启
systemctl start huo15-backend nginx
```

> 重要：v18.8 的 schema 变更不是破坏性的，可在不回滚 schema 的情况下回退应用版本。

## 六、故障排查

### 启动失败

```bash
journalctl -u huo15-backend -n 200 --no-pager
```

常见原因：

| 现象 | 排查方向 |
|------|---------|
| `psycopg2.OperationalError` | DB_URL 错误 / PostgreSQL 未启动 |
| `redis.exceptions.ConnectionError` | Redis 未启动 / 防火墙阻拦 |
| `Permission denied` | 文件归属错误（应为 huo15:huo15）|

### 性能问题

```bash
# 查询慢日志
psql -c "SELECT query, mean_exec_time FROM pg_stat_statements ORDER BY mean_exec_time DESC LIMIT 20;"

# Redis 命中率
redis-cli INFO stats | grep keyspace
```
