## 核心功能

**多规范自动识别**：根据文档标题和内容自动选择最佳格式模板

| 文档类型 | 自动识别关键词 | 适用场景 |
|---------|--------------|---------|
| 公文格式 | 默认（无关键词） | 正式函件、通知 |
| 合同格式 | 合同、协议、协议书 | 商业合同、合作协议 |
| 会议纪要 | 会议纪要、纪要 | 会议记录 |
| 技术方案 | 技术方案、实施方案、解决方案 | 项目方案 |
| 需求文档 | 需求、需求规格、SRS | 软件需求 |
| 工作报告 | 工作报告、周报、月报、述职报告 | 工作汇报 |

---

## 一、文档格式详解

### 1.1 公文格式（默认）

适用：正式函件、通知、请示

特点：
- 标题层次：一、→（一）→ 1.
- 字体：章=黑体16pt，节=楷体14pt，正文=仿宋12pt
- 页眉：LOGO+公司名+文档编号+密级
- 页脚：第X页/共Y页
- 版本历史表+审批签字区

### 1.2 合同格式

适用：商业合同、合作协议、保密协议

特点：
- 条款式结构：第一章 → 第一条
- 字体：宋体12-15pt
- 页眉：仅公司名（居中）
- 无版本历史，有审批签字区
- 甲乙方签署区

### 1.3 会议纪要格式

适用：会议记录、工作纪要

特点：
- 特殊标记：【会议主题】
- 字体：黑体14pt/仿宋12pt
- 无版本历史，无审批区
- 结构：时间/地点/人员/内容/决议

### 1.4 技术方案格式

适用：项目方案、实施方案、技术设计

特点：
- 阿拉伯数字编号：1. → 1.1 → 1.1.1
- 字体：黑体16pt/宋体12pt
- 公文式页眉页脚
- 版本历史表+审批签字区

### 1.5 需求文档格式

适用：软件需求规格说明书( SRS )

特点：
- 阿拉伯数字编号：1. → 1.1
- 字体：黑体16pt/宋体12pt
- 公文式页眉页脚
- 版本历史表+审批签字区
- 功能点列表格式

### 1.6 工作报告格式

适用：周报、月报、年报、述职报告

特点：
- 中文编号：一、 →（一）
- 字体：楷体14pt/仿宋12pt
- 公文式页眉页脚
- 无版本历史，有审批签字区
- 结构：背景/工作/问题/计划

## 二、标题识别规则

每个格式自动识别以下标题模式：

| 格式 | 一级标题 | 二级标题 | 三级标题 |
|-----|---------|---------|---------|
| 公文 | 第X章/第X节 | 一、二、三、 | （一）（二） |
| 合同 | 第X章 | 一、二、 | — |
| 会议纪要 | 【主题】 | 一、二、 | — |
| 技术方案 | 一、1. | 1.1 | 1.1.1 |
| 需求文档 | 一、1. | 1.1 | — |
| 工作报告 | 一、（一） | — | — |

## 三、API 调用

```python
from create_word_doc import create_word_doc

# 自动识别格式（推荐）
create_word_doc(
    output_path="文档.docx",
    title="技术方案：XXX系统",
    content="正文内容...",
    doc_number="HG-FA-2026-001",
    version="V1.0",
    classification="内部",
    author="辉火云管家·贾维斯"
)

# 显式指定格式
create_word_doc(
    output_path="合同.docx",
    title="采购合同",
    content="正文内容...",
    doc_format="合同"  # 可选：公文/合同/会议纪要/技术方案/需求文档/工作报告
)
```
